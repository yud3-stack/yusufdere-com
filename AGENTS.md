# AGENTS.md

## Project

YusufDere.com is the personal brand hub of Yusuf Dere.

This website is not a CV website.
This website is not a corporate portfolio.
It should feel like a premium, modern, minimal and timeless personal hub.

The site will be used for:
- personal brand
- projects
- journal
- gallery
- uses
- now page
- future startups and experiments

Instagram is only the showcase.
YusufDere.com is the main center.

## Identity

Instagram: @heyyud3  
Bio: Building the life I imagined.

## Tech Stack

Use:
- Next.js App Router
- React
- TypeScript
- Tailwind CSS
- Framer Motion
- Lucide React
- next-themes
- Sanity CMS
- Vercel deployment

Do not add unnecessary UI libraries.

## Design Direction

The design must feel:
- premium
- minimal
- modern
- calm
- cinematic
- timeless
- personal but not casual
- polished but not corporate

Visual references:
- Apple
- Vercel
- Linear
- Raycast
- Stripe

Main visual direction:
- dark theme first
- light theme later
- generous whitespace
- large typography
- subtle borders
- soft shadows only when necessary
- minimal glass effect
- no excessive gradients
- no cheap animations
- no colorful startup-template look

## CMS Content

Sanity CMS should manage:
- site settings
- projects
- journal posts
- gallery images
- now items
- uses items

Use CMS-ready structure even if placeholder content is used first.

## Coding Rules

- Use Server Components by default.
- Use Client Components only when needed for interactivity, theme switching or animation.
- Keep components small, reusable and clean.
- Use semantic HTML.
- Use next/image for images.
- Optimize for performance.
- Use accessible buttons, links and headings.
- Avoid hardcoded repeated content.
- Keep content in Sanity or typed config files.
- Do not create messy one-file pages.
- Do not change the design direction without explicit approval.
- Do not make the website look like a generic portfolio template.

## Motion Rules

Use Framer Motion carefully.

Allowed:
- subtle fade in
- slight upward entrance
- soft hover transitions
- page-level smooth reveal

Avoid:
- bouncing
- excessive parallax
- aggressive scaling
- flashy transitions

## Important

Before implementing large features, first plan:
- component structure
- data structure
- responsive behavior
- CMS schema

Premium feeling is more important than adding many features.
